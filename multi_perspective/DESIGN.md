# 多视角分析模块设计方案

> 借鉴 ai-hedge-fund 的多 Agent 投票机制，为 stock-analyst-skill 增加投资风格多视角分析能力。

## 一、设计理念

### 核心差异：不是让 LLM "模仿巴菲特"，而是用 LLM 做"风格化解读"

ai-hedge-fund 的问题：同一个 LLM + 不同 prompt，风格 Agent 间同质化严重。
我们的方案：**本地量化计算为主（零 Token），LLM 只在最终生成文本时做风格化润色**。

```
                    ┌─────────────────────────────┐
                    │     市场数据（已有）           │
                    │  行情 / K线 / 财报 / 资金流   │
                    └──────────┬──────────────────┘
                               │
                    ┌──────────▼──────────────────┐
                    │   本地量化计算（零 Token 成本） │
                    │  技术指标 / 估值 / 资金流 /     │
                    │  情绪指数 / 形态识别 / 缠论    │
                    └──────────┬──────────────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                ▼
    ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
    │  价值派视角   │  │  成长派视角   │  │  技术派视角   │

                  └────────┬────────┘

                ┌──────────────────┐

                │  ViewAggregator  │
                │                  │

                │  报告输出         │

                └──────────────────┘
```

### 关键设计原则

1. **零 Token 核心计算**：5个视角全部用已有数据本地算，不调 LLM
2. **风格化靠规则而非角色扮演**：每个视角有明确的关注指标和评分规则
3. **LLM 只做最后一步**：将结构化结果转化为自然语言报告时才消耗 Token
4. **与现有体系无缝集成**：复用 `signals/scoring.py` 的信号体系

## 二、5个投资视角定义

### 视角1：价值派（Value Perspective）

**投资哲学**：寻找价格低于内在价值的标的，注重安全边际。

| 关注指标 | 数据来源 | 判定规则 |
|---------|---------|---------|
| PE 估值 | 行情数据 | < 15 看多，15-30 中性，> 50 看空 |
| PB 估值 | 行情数据 | < 1.5 看多，1.5-3 中性，> 5 看空 |
| 股息率 | 基本面数据 | > 4% 看多，2-4% 中性，< 1% 看空 |
| ROE | 财报数据 | > 15% 看多，8-15% 中性，< 5% 看空 |
| 营业利润率 | 财报数据 | > 20% 看多，10-20% 中性 |

**输出**：opinion + confidence(0-1) + reasons[]

### 视角2：成长派（Growth Perspective）

**投资哲学**：关注未来增长潜力，愿意为成长支付溢价。

| 关注指标 | 数据来源 | 判定规则 |
|---------|---------|---------|
| 营收增长率 | 财报对比 | > 20% 看多，10-20% 中性，< 5% 看空 |
| 净利润增长率 | 财报对比 | > 25% 看多，10-25% 中性，< 0% 看空 |
| 行业景气度 | 新闻/板块数据 | 高景气 看多，中性，低迷 看空 |
| 毛利率趋势 | 财报对比 | 上升 看多，持平 中性，下降 看空 |
| 研发投入比 | 财报数据 | > 10% 看多（科技股），5-10% 中性 |

**输出**：opinion + confidence + reasons[]

### 视角3：技术派（Technical Perspective）

**投资哲学**：价格包含一切信息，趋势是朋友。

| 关注指标 | 数据来源 | 判定规则 |
|---------|---------|---------|
| 均线系统 | K线数据计算 | 多头排列 看多，空头 看空 |
| MACD | K线数据计算 | 金叉 看多，死叉 看空 |
| RSI | K线数据计算 | < 30 超卖看多，> 70 超买看空 |
| KDJ | K线数据计算 | J 值超卖 看多，超买 看空 |
| 量价配合 | K线数据计算 | 放量涨 看多，缩量跌 看空 |
| 支撑/阻力位 | K线数据计算 | 接近支撑 看多，接近阻力 看空 |

**输出**：opinion + confidence + reasons[]（复用 `signals/scoring.py`）

### 视角4：逆向派（Contrarian Perspective）

**投资哲学**：当市场一致看多时要警惕，极度恐慌时寻找机会。

| 关注指标 | 数据来源 | 判定规则 |
|---------|---------|---------|
| 市场情绪指数 | `sentiment_index.py` | < 20 逆向看多，> 80 逆向看空 |
| 融资余额变化 | 资金面数据 | 融资余额骤降 逆向看多，骤升 逆向看空 |
| 机构一致性 | 新闻/研报 | 一致看多时降 confidence |
| 涨停/跌停板 | 行情数据 | 跌停潮 逆向看多，涨停潮 逆向看空 |
| 新高/新低比 | 大盘数据 | 创新低股票占比高 逆向看多 |

**输出**：opinion + confidence + reasons[]

### 视角5：风控派（Risk Perspective）

**投资哲学**：不亏钱比赚钱重要，控制回撤是第一要务。

| 关注指标 | 数据来源 | 判定规则 |
|---------|---------|---------|
| 最大回撤 | K线数据计算 | > 20% 高风险，10-20% 中风险，< 10% 低风险 |
| 波动率 (ATR) | K线数据计算 | 高波动 降 confidence |
| 连涨/连跌天数 | K线数据计算 | > 5天连涨 减分，> 5天连跌 加分(反弹预期) |
| 成交量异常 | K线数据计算 | 天量后缩量 看空信号 |
| 盈亏比 | 仪表盘数据 | < 1.5 不建议操作 |

**特殊行为**：风控派不输出 opinion，只输出风险等级和约束条件。

**输出**：risk_level(low/medium/high) + max_position(%) + warnings[] + constraints{}

### 投票逻辑

```python
# 每个视角输出：
# opinion: 'bullish' | 'bearish' | 'neutral'
# confidence: 0.0 - 1.0
# weight: 预设权重

# 加权计算：
bullish_score = Σ(opinion=='bullish' × confidence × weight) for each view
bearish_score = Σ(opinion=='bearish' × confidence × weight) for each view

# 风控一票否决：
if risk_view.risk_level == 'high':
    final_opinion = 'neutral'  # 高风险时强制观望
    final_confidence *= 0.5    # 降低置信度

# 分歧检测：
views_opinions = [v.opinion for v in views]
if bullish_count >= 3 and bearish_count >= 2:
    flag = "⚠️ 视角分歧较大"
```

## 五、与现有模块的集成点

```
                    现有模块                          新增模块
                    ──────                          ──────
scoring.py ─── 信号体系 ──────────────────► 技术派视角（复用 Signal 类）
sentiment_index.py ─── 情绪指数 ─────────► 逆向派视角（复用 SentimentResult）
dashboard_template.py ─── 仪表盘 ──────► 新增"多视角分析"区块
stock_analyzer.py ─── 分析入口 ─────────► 集成 multi_perspective.analyze()
cost_tracker ─── 成本追踪 ────────────► 仅 LLM 润色步骤计入
data_adapter ─── 数据获取 ────────────► 所有数据复用，无新增依赖
```

### 最小改动集成

```python
# stock_analyzer.py 中新增

    result['multi_perspective'] = perspective
```

## 六、Token 成本估算

| 步骤 | 方式 | Token 消耗 |
|------|------|-----------|
| 5个视角计算 | 本地 Python | 0 |
| 投票聚合 | 本地 Python | 0 |
| 结构化报告输出 | 模板拼接 | 0 |
| LLM 风格化润色（可选） | API 调用 | ~800-1500 tokens/次 |
| **总计（不润色）** | - | **0** |
| **总计（含润色）** | - | **~1000 tokens** |

对比 ai-hedge-fund 的 19 个 Agent 每只股票约 20,000+ tokens，我们的方案成本降低 95%+。
